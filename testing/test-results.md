# Test results

You can view all your test results and take action on them in one place.

1. Go to `Testing` > `Test results`
   <img width="818" alt="Test results" src="https://user-images.githubusercontent.com/91221068/230916382-382479db-a15d-4579-bd0e-5371db6f595b.png">

2. Use `Scheduled tests` section to find daily scheduled tests. All the past test runs are in the `Previous tests` section. 

3. To see all test runs in a category, click on the heading `All tests`
   <img width="1016" alt="See all test runs" src="https://user-images.githubusercontent.com/91221068/230916443-2571081c-0e7e-4614-9dc2-acdf47e92559.png">


4. To see results for a particular test, select the test run name. 
   <img width="1228" alt="Test failures over time" src="https://user-images.githubusercontent.com/91221068/230916473-c18f8bbc-64d3-4c38-815a-9d62f07ce094.png">


5. You can see a list of all test failures with respective `Severity` marked. 
   <img width="877" alt="Test failures" src="https://user-images.githubusercontent.com/91221068/230916485-2f127392-75e0-437f-abc3-b780bb83f4c3.png">


6. Click on any entry to know the issue description
   <img width="976" alt="Description" src="https://user-images.githubusercontent.com/91221068/230916532-d594aa07-d57e-4f65-896a-34731b84e21b.png">


7. Select the `Original` tab to know what was the original request. You can click on the `Copy` icon on the top right to copy it as a `curl` or `Burp` request.
   <img width="986" alt="Original" src="https://user-images.githubusercontent.com/91221068/230916536-d03385cf-8890-411e-8830-223b75e0d5aa.png">


8. Select the `Attempt` tab to see what malicious payload(s) Akto sent to find out the vulnerability. Copy button is available here as well. You might want to try it on your own or send it to your dev team.
   <img width="988" alt="Attempt" src="https://user-images.githubusercontent.com/91221068/230916548-4812137e-7198-4cee-832f-d49e47d4b911.png">
